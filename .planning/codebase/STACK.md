# Technology Stack

**Analysis Date:** 2026/04/26

## Languages

**Primary:**
- C++17 - All application source code (`*.cc`, `*.hpp`)
- JavaScript - KWin script for KDE Plasma window tracking (`Platform/Linux/kwin_script.js`)
- JavaScript - GNOME Shell extension (`Platform/Linux/gnome_script/extension.js`)

**Build Scripts:**
- Bash - Build scripts (`bundle-default.sh`, `find_dlls.sh`, `bin2cpp.sh`)
- XML - Mascot behavior/action definitions (`DefaultMascot/behaviors.xml`, `DefaultMascot/actions.xml`)
- CMake - Submodule builds (`libshijima/CMakeLists.txt`, `libshimejifinder/CMakeLists.txt`)

## Runtime

**Environment:**
- Native compiled binary (no runtime interpreter)
- Qt6 application framework (GUI event loop via `QApplication`)

**Build System:**
- GNU Make (`Makefile`, `common.mk`, `Platform/Makefile`)
- CMake 3.14+ (for submodules: `libshijima/`, `libshimejifinder/`)

**Package Manager:**
- No package manifest at project root
- Dependencies installed via system package managers:
  - **macOS:** MacPorts (`sudo port install qt6-qtbase qt6-qtmultimedia pkgconfig libarchive`)
  - **Linux:** apt (`libarchive-dev`, Qt6 packages via pkg-config)
  - **Windows:** MinGW64 via Docker (`dev-docker/Dockerfile`)

## Frameworks

**Core:**
- Qt6 - GUI framework, cross-platform abstraction
  - `Qt6::Widgets` - Main UI components (`QMainWindow`, `QWidget`, `QDialog`)
  - `Qt6::Core` - Core utilities (`QString`, `QMap`, `QSettings`, `QTimer`)
  - `Qt6::Gui` - Painting, events (`QPainter`, `QMouseEvent`, `QImage`)
  - `Qt6::Concurrent` - Threading support
  - `Qt6::Multimedia` - Sound effects via `QSoundEffect` (optional, `SHIJIMA_USE_QTMULTIMEDIA` flag)
  - `Qt6::DBus` - Linux DBus integration (KWin/GNOME communication)

**HTTP Server/Client:**
- cpp-httplib v0.19.0 (header-only, via `cpp-httplib/`) - HTTP API server on `127.0.0.1:32456`

**Key Dependencies:**

**Critical:**
- `libshijima` (git submodule) - Mascot behavior engine, produces `libshijima.a`. Uses Duktape for scripting, RapidXML for XML parsing.
- `libshimejifinder` (git submodule) - Archive extraction for mascot .zip files. Produces `libshimejifinder.a`. Depends on `unarr` and `libarchive`.
- `cpp-httplib` (header-only) - HTTP server/client library embedded at `cpp-httplib/httplib.h`
- `libarchive` (system) - Archive format support for mascot import
- `unarr` (bundled via libshimejifinder) - RAR/7z/ZIP extraction engine. Produces `libunarr.so.1` / `libunarr.1.dylib`

**Platform-Specific:**
- Linux: X11 (`libxcb`), DBus (`Qt6DBus`), KWin scripting
- macOS: AppKit.framework, ApplicationServices.framework, libobjc
- Windows: ws2_32 (Winsock2), MinGW runtime

## Configuration

**Environment:**
- No `.env` file or environment variable configuration required
- Build-time configuration via Makefile variables:
  - `CONFIG` — `release` (default, `-O3 -DNDEBUG`) or `debug` (`-g -O0`)
  - `SHIJIMA_USE_QTMULTIMEDIA` — `1` (default, enables sound) or `0`
  - `PREFIX` — `/usr/local` (install path)
  - `QMAKE` — `qmake` (Qt6) or `qmake-qt5` (Qt5 override)

**Build:**
- `common.mk` — Shared build configuration (C++17 standard, Qt6, platform detection)
- `Makefile` — Top-level build rules, source list, publish targets
- `Platform/Makefile` — Platform library build dispatcher
- `Platform/{Linux,macOS,Windows}/Makefile` — Per-platform build rules
- `Platform/Linux/Makefile` — Linux-specific: KWin/GNOME script compilation, DBus

**Generated Sources:**
- `DefaultMascot.cc` — Auto-generated by `bundle-default.sh` from 46 PNG frames + XML
- `licenses_generated.hpp` — Auto-generated for in-app license display
- `kwin_script.c` / `gnome_script.c` — C arrays from minified JS/ZIP (Linux only)

**Build Modes:**
| Mode | CXXFLAGS | CMake |
|------|----------|-------|
| `release` | `-O3 -DNDEBUG` | `-DCMAKE_BUILD_TYPE=Release` |
| `debug` | `-g -O0` | `-DCMAKE_BUILD_TYPE=Debug` |

## Platform Requirements

**Development:**
- **Linux:** Qt6 (Widgets, Core, Gui, Concurrent, DBus, Multimedia), libarchive-dev, pkg-config, make, cmake, g++
- **macOS:** Xcode command-line tools, MacPorts with qt6 packages, libarchive
- **Windows:** MinGW64 toolchain (via Docker), Fedora 42 base

**Production:**
- **Linux:** x86_64 or arm64. Distributed as AppImage or raw binary + `libunarr.so.1`
- **macOS:** arm64 (Apple Silicon). Distributed as `.app` bundle
- **Windows:** x86_64. Distributed as `.exe` + DLLs
- **GNOME:** Shell extension auto-installed on first run; requires logout/login
- **KDE:** KWin script auto-installed; transparent to user

## Tooling

**Build Tools:**
- `pkg-config` - Qt6 and libarchive discovery on Linux
- `qmake` - Qt6 build configuration resolution
- `cmake` - Submodule builds
- `strip` - Binary size optimization in release builds
- `hexdump` / `sed` - Asset bundling (`bin2cpp.sh`)
- `yarn` / `minify` - GNOME extension JS minification (Linux only)
- `python3` - GNOME extension ZIP packaging
- `linuxdeploy` + `linuxdeploy-plugin-qt` - Linux AppImage generation
- `macdeployqt` - macOS `.app` bundle creation
